# DESIGN.md

这个文件只记录稳定设计原则与设计文档入口，不复写完整 design memo。

## Stable Principles

- Humans steer, agents execute
- repo-local knowledge 应该是 system of record
- 入口文档要短，深层文档要可发现
- runtime vault 与 harness repo 的边界必须清晰
- 机械检查优先于口头约定
- viewer / plugin 是次级界面，不是 canonical maintenance entry
